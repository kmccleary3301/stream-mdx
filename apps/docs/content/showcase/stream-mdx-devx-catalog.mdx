## Quickstart

Stream-MDX ships as modular packages plus a convenience bundle. Choose the level of control you need.

```bash
npm install stream-mdx
# or the modular packages:
npm install @stream-mdx/react @stream-mdx/worker @stream-mdx/core @stream-mdx/plugins
```

Minimal client usage with the worker and the React component:

```tsx
import { StreamingMarkdown } from "@stream-mdx/react";

export default function Demo() {
  return (
    <StreamingMarkdown
      initialContent="# Hello"
      content="# Hello\nThis updates as text streams in."
    />
  );
}
```

## Packages and Modularity

Each package is small, focused, and composable.

- @stream-mdx/core - parsing, inline tokens, streaming diff logic, patch coalescing
- @stream-mdx/worker - the worker runtime (parsing, highlighting, MDX compile parity)
- @stream-mdx/react - React renderer, worker orchestration, metrics
- @stream-mdx/plugins - optional plugins (regex, html, math, tables, etc)
- stream-mdx - convenience bundle that re-exports the common surface

Recommended paths:

- Simple app: use stream-mdx or @stream-mdx/react + @stream-mdx/worker
- Custom editor: use @stream-mdx/core directly for inline parsing and patches
- Server-only render: use @stream-mdx/worker in server compile mode

## Architecture and Streaming Modes

Stream-MDX can compile MDX in a worker or on the server. Both modes share the same compiler helper,
so outputs match.

```tsx
<StreamingMarkdown
  mdxCompileMode="worker"
  content={streamedText}
  mdxComponents={{ Preview, YouTube, Image }}
/>
```

Server compile mode example:

```tsx
<StreamingMarkdown
  mdxCompileMode="server"
  content={streamedText}
  mdxComponents={{ Preview, YouTube, Image }}
/>
```

**Note:** Worker mode is preferred for max throughput and minimal UI thread work.

## Feature Flags and Doc Plugins

Doc plugins allow you to enable or disable classes of behavior.

```tsx
<StreamingMarkdown
  docPlugins={{
    math: true,
    html: true,
    mdx: true,
    tables: true,
    footnotes: true,
    formatAnticipation: { inline: true, mathInline: true },
    liveCodeHighlighting: false,
  }}
/>
```

**Tip:** Keep features minimal for untrusted content. Enable additional parsing only when needed.

## Format Anticipation

Format anticipation is optional and gives you streamdown-like partial rendering: bold, italic, or math
segments can appear before their closing delimiters arrive.

```tsx
docPlugins={{
  formatAnticipation: {
    inline: true,
    mathInline: true,
    mathBlock: true,
    html: true,
    mdx: true,
    regex: true,
  },
}}
```

Streaming examples (input on the left, expected render on the right):

```text
*This is incomplete text  ->  <em>This is incomplete text</em>
**This is incomplete text ->  <strong>This is incomplete text</strong>
$$x = y               ->  block math renders, until newline threshold
{cite:5              ->  citation placeholder auto-closes to {cite:5}
```

## Markdown Basics

Basic markdown is always supported, even with all plugins disabled.

```md
# Heading 1
## Heading 2
### Heading 3

Paragraphs are separated by blank lines.

**Bold**, *italic*, and ~~strike~~ are supported.

Inline code uses backticks: `like this`.
```

## Code Blocks and Syntax Highlighting

Stream-MDX uses Shiki for syntax highlighting. You can toggle live highlighting while streaming.

````md
```ts
type Config = {
  mode: "worker" | "server";
};
```
````

```tsx
<StreamingMarkdown
  docPlugins={{ liveCodeHighlighting: true }}
/>
```

**Note:** Live highlighting costs more; keep it off for max throughput.

## Math (Inline and Block)

Inline math uses single dollars. Block math uses double dollars.

```md
Inline: $E = mc^2$

$$
\int_0^\infty e^{-x^2} dx = \frac{\sqrt{\pi}}{2}
$$
```

Custom delimiters can be supported with an inline regex plugin (example below).

## HTML and MDX

Inline HTML and block HTML are supported and sanitized.

```md
<div class="p-2 border">Raw HTML block</div>
<kbd>Inline kbd</kbd>
```

MDX components are supported in both worker and server compile modes.

```mdx
<Preview>
  <PreviewExample />
</Preview>

<YouTube videoId="dQw4w9WgXcQ" caption="Example Video" />

<Image src="https://avatar.vercel.sh/sylph" alt="Avatar" caption="Example image" />
```

## Custom MDX Components

Register MDX components and stream-mdx will resolve them during compile.

```tsx
const mdxComponents = { Preview, YouTube, Image, Callout };
<StreamingMarkdown mdxComponents={mdxComponents} />
```

Components can be used inline and nested.

## Custom Regex Plugins

Regex plugins let you add domain syntax without forking the parser.

```ts
import { InlineParser } from "@stream-mdx/core";

const parser = new InlineParser();
parser.addRegexPlugin({
  id: "citation",
  priority: 30,
  re: /\{cite:([^}]+)\}/g,
  toNode: (match) => ({ kind: "citation", id: match[1] }),
  anticipation: {
    start: /\{cite:/,
    full: /\{cite:[^}]+\}/,
    append: "}",
  },
});
```

Example usage: `See {cite:smith2024} for details.`

## Custom Renderers and ShadCN Overrides

Map HTML tags to design-system components for a polished look. This does not break streaming.

```tsx
const components = {
  table: (props) => <Table {...props} />,
  thead: (props) => <TableHeader {...props} />,
  tbody: (props) => <TableBody {...props} />,
  tr: (props) => <TableRow {...props} />,
  th: (props) => <TableHead {...props} />,
  td: (props) => <TableCell {...props} />,
  blockquote: (props) => <Blockquote {...props} />,
};

<StreamingMarkdown components={components} />
```

## Styling and Theming

Use your existing design tokens. The renderer emits stable classes (mdx-*, markdown-*).

```css
.markdown-root {
  font-size: 16px;
  line-height: 1.6;
}
.markdown-root code {
  background: var(--muted);
  padding: 0 0.2rem;
}
.markdown-root table {
  width: 100%;
}
```

You can also pass className or style to wrap content.

## Performance and Backpressure

Backpressure keeps UI responsive when high-frequency updates arrive.

```tsx
<StreamingMarkdown
  backpressure={{
    maxPendingChars: 4000,
    maxPendingBlocks: 50,
    updateIntervalMs: 16,
  }}
/>
```

## Metrics and Telemetry

StreamingMarkdown can emit metrics about parse and render time.

```tsx
<StreamingMarkdown
  onMetrics={(metrics) => {
    console.log(metrics.parseMs, metrics.diffMs, metrics.shikiMs);
  }}
/>
```

## Worker Hosting and CSP

For strict CSPs, host the worker yourself.

```bash
npm -w @stream-mdx/worker run build:hosted
# copy dist/hosted/markdown-worker.js to your public/ directory
```

```tsx
<StreamingMarkdown workerOptions={{ workerUrl: "/workers/markdown-worker.js" }} />
```

## Security Model

HTML is sanitized. Allowlists are applied for tags and attributes. If you enable raw HTML, keep
sanitizers on and avoid unsafe plugins.

```ts
const features = { html: true };
// Sanitization is handled internally for HTML and code highlighting.
```

## Testing and Regression Harness

Stream-MDX ships with regression tests for patch coalescing, backpressure, and streaming edge cases.

```bash
npm run markdown-v2:test:packages
npm run markdown-v2:nightly
```
